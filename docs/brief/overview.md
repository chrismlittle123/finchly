- "Ask less, know more"

## The Problem

Status meetings waste everyone's time:

| Pain Point | Reality |
|------------|---------|
| People report what tools already know | "I merged 12 PRs" â€” GitHub knows this |
| Same questions every week | "What did you work on?" â€” repetitive, generic |
| Easy to BS or ramble | No accountability, no data to verify |
| Managers chase updates | Slack pings, email follow-ups, meetings |
| Takes 15-30 minutes | Per person, per week â€” it adds up |

---

## The Solution

Brief pulls from your team's tools, identifies what it can't figure out, and asks only targeted questions.

### The Psychology

| Old way (status meetings) | Brief way |
|---------------------------|-----------|
| "Tell me everything" | "Tell me what I can't see" |
| Feels like surveillance | Feels like assistance |
| Takes 15-30 min | Takes 3-4 min |
| Easy to BS | Hard to BS (Brief has the data) |
| Accountability is in the meeting | Accountability is async + visible |
| Manager chases you | Brief finds the right moment |
